# API ارسال محصولات به پارتجو

**نسخه 1.2 – تیر 1404**

برای ارسال محصولات خود به پارتجو تنها لازم است API زیر را با `Post method` صدا کرده و مقادیر زیر را به عنوان `body` ارسال نمائید.

`https://partjoo.com/partjoo/apiv1`

```json
{ 
  "route": "crawler/addProductsToPartjoo",
  "content": {
    "domain": "",
    "allLinks": [],
    "products": []
  }
}
```

شما می‌توانید در آرایه‌های مربوط به محصولات، تمام محصولات خود را قرار داده یا برای کم کردن حجم دیتای ارسالی با استفاده از یک تابع بازگشتی محصولات را طی چند مرحله برای ما ارسال کنید. البته در نظر داشته باشید جهت عدم بروز مشکل؛ همواره حداکثر **100 محصول** را در یک درخواست ارسال نمائید.

### توضیح کلیدها:

- جدول شماره1: توضیح کلیدها

| ردیف | عنوان کلید   | نوع داده | اجباری؟ | توضیح                                                                                                                                                                                                                                                        |
| ---- | ------------ | :------- | :------ | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1    | title        | string   | بله     | عنوان محصول (در برخی سایت‌ها این عنوان دقیقاً با partNumber یا partName یکی می‌باشد؛ در این صورت نیازی به ارسال partNumber/partName نمی‌باشد.)                                                                                                                   |
| 2    | url          | string   | بله     | آدرس کامل صفحه‌ی محصول                                                                                                                                                                                                                                        |
| 3    | availability | string   | بله     | وضعیت موجود بودن محصول. اگر سایت فقط موجود یا ناموجود بودن را اعلام می‌کند و تعداد موجودی ندارد، این فیلد استفاده شود. مقادیر قابل قبول “1” (موجود) و “-1” (ناموجود) می‌باشد.                                                                                  |
| 4    | image        | string   | خیر     | آدرسی که تصویر اصلی محصول از آن خوانده می‌شود                                                                                                                                                                                                                 |
| 5    | price        | number   | خیر     | قیمت محصول در حالت فروش تکی؛ توجه داشته باشید که قیمت همواره باید به ریال بوده و درصورت نیاز قبل از ارسال اصلاح شود تا قیمت در موتور جستجو اشتباه نمایش داده نشود.                                                                                           |
| 6    | bulkPrices   | Json     | خیر     | در صورت وجود قیمت‌های خرید عمده، می‌توان از این کلید به این صورت استفاده کرد:`"bulkPrices": [{ "quantity": 10, "price": 950000 }, { "quantity": 100, "price": 900000 }]`                                                                                       |
| 7    | unit         | string   | خیر     | یکی از مقادیر: ”toman” “rial” “dollar” “yuan” (در صورت عدم ارسال پیش‌فرض ریال در نظر گرفته می‌شود)                                                                                                                                                             |
| 8    | stock        | number   | خیر     | تعداد موجودی محصول – اگر تعداد نامشخص بود یا فقط موجود/ناموجود بودن محصول مشخص است، مقدار 0 قرار داده شود و از فیلد availability برای وضعیت استفاده شود.                                                                                                     |
| 9    | condition    | string   | خیر     | شرایط قطعه طبق حالات جدول شماره 2 که در ذیل این جدول آورده شده است.<br />در صورتی که تمامی قطعات شما original است، بهتر است همواره این فیلد را “new” قرار دهید. در غیر این صورت ممکن است در سایت وضعیت محصول شما "نامعلوم" یا کلا بدون وضعیت نمایش داده شود. |
| 10   | partName     | string   | خیر     | در برخی سایت‌ها، علاوه بر عنوان برای محصول که ممکن است فارسی و یا اصطلاح بازاری محصول باشد، یک عنوان انگلیسی و یا متفاوت از آن عنوان نیز وجود دارد، در این حالت این مقدار را در این فیلد قرار دهید.                                                           |
| 11   | partNumber   | string   | خیر     | در برخی سایت‌ها علاوه بر عنوان محصول؛ part number محصول نیز درج شده است. ممکن است برخی سایت‌ها این مقدار را "نام کارخانه‌ای قطعه" و یا مواردی مشابه بنامند.                                                                                                     |
| 12   | description  | string   | خیر     | توضیحات محصول. به طولانی نشدن بیش از اندازه‌ی این فیلد توجه داشته باشید.                                                                                                                                                                                      |

- جدول شماره2: توضیح حالت های مختلف مقدار ارسالی برای condition

| مقدار ارسالی | توضیح                                                                            |
| ------------ | -------------------------------------------------------------------------------- |
| new          | قطعه نو و اورجینال                                                               |
| oem          | قطعه تولید شده تحت لایسنس                                                        |
| copy         | قطعه کپی (قطعه ای که مشابه قطعه ی اصلی ولی بدون لایسنس در چین و.. تولید شده است) |
| renew        | قطعه ی نوسازی شده / قطعه ی دمونتاژ شده از روی برد                                |
| used         | قطعه ی دست دوم                                                                   |
| nos          | قطعات خیلی قدیمی که البته نو (استفاده نشده) هستند                                |

### نکات:

1. لطفاً **دقیقاً** از همین کلیدها استفاده کنید. همانطور که مشاهده می‌شود عنوان کلیدها همگی به صورت `Lower camelCase` می‌باشند.
2. در حال حاضر نیازی به قرار دادن مقداری در فیلد `allLinks` نمی‌باشد. ولی در ارسال این فیلد را به همین صورت خالی قرار دهید.
3. در نظر بگیرید که در هنگام ویرایش محصولات در سرور خود می‌توانید بار دیگر این API را صدا کرده و فقط کالای ویرایش شده را در آرایه قرار دهید.
4. توجه داشته باشید که مقدار دامنه در `body` را باید از تیم فنی پارتجو دریافت کرده و عیناً همان را قرار دهید.
